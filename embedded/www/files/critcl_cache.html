Description

Welcome to the C Runtime In Tcl, CriTcl for short, a system to build C extension packages for Tcl on the fly, from C code embedded within Tcl scripts, for all who wish to make their code go faster.

This document is the reference manpage for the critcl::cache package. This internal utility package provides the commands used by the core to manage the result cache directory. Its intended audience are mainly developers and maintainers of critcl itself. Regular users have no need for the commands of this package.

This package resides in the Core Package Layer of CriTcl.

API

::critcl::cache::append file content

This command appends the content to the file in the result cache. The result of the command is the full path of the modified file.

::critcl::cache::clear ?pattern...?

This command removes all files matching any of the patterns from the result cache. If no pattern is specified it defaults to *, i.e. removing all files found in the cache. The result of the command is the empty string.

::critcl::cache::copy2 src ?dst?

This command copies the src path into the result cache as dst. If dst is not specified it defaults to the last part of the src path. The result of the command is the full path of dst.

::critcl::cache::def path

This command sets the location of the result cache to path. The result of the command is the empty string.

::critcl::cache::get ?path?

This command returns the location of the result cache, if called without a path specified. With path specified its full path within the result cache is returned. Note that the path does not have to exist.

::critcl::cache::write

This command stores the content in the file in the result cache, replacign any previous content, if any. The result of the command is the full path of the modified file.
